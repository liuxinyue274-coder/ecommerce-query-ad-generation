项目背景与研究意义

从搜索广告的“召回-排序”结果中进一步生成可投放、可解释、可校验的创意文案。

问题背景

传统电商搜索广告文案往往依赖标题拼接或模板生成，难以表达用户 Query 背后的真实场景、偏好和购买关注点。大语言模型具备意图理解和自然语言生成能力，但需要事实约束与质量控制。

研究意义

本系统将商品证据、用户画像、风格控制和多候选排序结合起来，在避免虚构卖点的前提下提升广告文案的相关性、自然度、多样性和可解释性。

系统演进的四个核心阶段

从规则模板到 V5 模块化链路，逐步增强可控性、可解释性和展示能力。

PHASE 01

数据与检索基线

整理 KuaiSearch lite 数据，构建 Query、商品证据、用户行为与排序信号的基础输入。

PHASE 02

规则化创意生成

基于商品标题、品牌、类目和点击购买信号生成可运行的广告文案初版。

PHASE 03

LLM Provider 接入

引入 local_fake 与 deepseek_chat 的 provider 抽象，并预留 sft_local 模型入口。

PHASE 04

V5 可解释链路

拆分意图增强、证据选择、Prompt、解析、校验、排序、重写和最终文案输出模块。

V5 系统流程

模块化链路支持逐步调试、失败回退和离线评测。

Query→ query_normalization→ intent_enricher→ evidence_selector→ user_profile_builder→ style_retriever→ prompt_builder→ llm_provider→ output_parser→ copy_validator→ copy_ranker→ copy_rewriter→ final_ad_copy

自动评测指标

从语义质量、流畅度、事实约束、多样性和系统稳定性多个维度比较生成方案。

BERTScore衡量生成文案与参考文案的语义一致性。
PPL衡量语言流畅度和困惑度。
证据覆盖率检查文案是否覆盖商品、品牌、类目和卖点证据。
Distinct-n统计 n-gram 多样性，避免模板化重复。
多样性分析候选文案表达角度与品类覆盖。
Fallback Rate统计 provider 失败、解析失败和本地回退比例。
Provider 对比比较 local_fake、deepseek_chat 与 sft_local 的质量和延迟。
